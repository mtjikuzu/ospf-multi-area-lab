## Description

```
Complete hands-on multi-area OSPF configuration lab using 5 Cisco IOL routers in ContainerLab. Watch every command typed in real time — no slides, no fluff.

Lab configs & topology: https://github.com/mtjikuzu/ospf-multi-area-lab

What you'll see:
  00:00  Intro — What is OSPF?
  00:40  Topology Overview (5 routers, 3 areas, 2 ABRs)
  01:15  Configure R1 — Backbone core router (Area 0)
  03:14  Configure R2 — ABR bridging Area 0 → Area 1
  05:26  Configure R3 — ABR bridging Area 0 → Area 2
  07:28  Configure R4 — Distribution router (Area 1)
  09:26  Configure R5 — Distribution router (Area 2)
  11:34  Convergence — Watch adjacencies form in real time
  12:36  Verify OSPF Neighbors (show ip ospf neighbor)
  14:30  Verify Routing Table (show ip route ospf)
  15:18  Lab Complete — All routes installed
  15:42  Summary & Closing

Topology:
  • R1, R2, R3 — Area 0 (backbone), full mesh
  • R2 → R4 — Area 1 (distribution)
  • R3 → R5 — Area 2 (distribution)
  • R4 ↔ R5 — Cross-link for redundancy
  • 7 routers/switches, 3 OSPF areas, 14 adjacencies

Tools used:
  • Cisco IOL (IOS on Linux) via ContainerLab
  • Remotion for animated intro/outro
  • Everything runs on a single Linux server

Key concepts covered:
  ✓ Multi-area OSPF design (backbone + non-backbone areas)
  ✓ Router ID assignment via Loopback0
  ✓ Point-to-point OSPF network type
  ✓ Area Border Routers (ABR) — R2 and R3
  ✓ Inter-area (O IA) routes vs intra-area (O) routes
  ✓ OSPF neighbor adjacency verification
  ✓ OSPF routing table analysis

This is a lab walkthrough — not a lecture. Every command is typed live on real routers. Pause and follow along at your own pace.

Music: "Wibbbly" by Shane Ivers — https://www.silvermansound.com
```
